## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router pages and routes
- Contains: Page components (`.tsx` files named `page.tsx`), dynamic route segments, layout wrapper
- Key files: `page.tsx` (home), `projects/page.tsx` (gallery), `projects/[slug]/page.tsx` (detail), `layout.tsx` (root wrapper)

**`src/components/`:**
- Purpose: Reusable React components
- Contains: Layout components (Navigation, Footer), page sections (Hero, Tagline, ProjectGrid, Philosophy), UI primitives (InsleyGrid)
- Key files: `layout/Navigation.tsx`, `sections/ProjectGrid.tsx`, `ui/InsleyGrid.tsx`

**`src/components/layout/`:**
- Purpose: Components that frame pages
- Contains: Navigation (header), Footer (brand, address, links)
- Key files: `Navigation.tsx`, `Footer.tsx`

**`src/components/sections/`:**
- Purpose: Major page sections that compose homepage and detail pages
- Contains: HeroSection, TaglineSection, ProjectGrid, PhilosophyBand, ProcessSection
- Key files: All `.tsx` files in this directory

**`src/components/ui/`:**
- Purpose: Reusable visual primitives
- Contains: InsleyGrid (SVG grid), ArchDrawing (architectural drawing placeholders)
- Key files: `InsleyGrid.tsx`, `ArchDrawing.tsx`

**`src/lib/`:**
- Purpose: Utilities and data
- Contains: Project type definitions, project array, lookup functions
- Key files: `projects.ts` (everything in this layer)

**`src/styles/`:**
- Purpose: Global styles and design system
- Contains: CSS custom properties (design tokens), Tailwind directives, utility classes
- Key files: `globals.css` (single source of truth for design)

**`public/images/`:**
- Purpose: Static project photography
- Contains: Subdirectories named by project ID (e.g., `house-01/`, `house-02/`)
- Key files: Project images referenced in `Project.coverImage` and `Project.images` arrays

**`.planning/codebase/`:**
- Purpose: Generated codebase analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Key files: All markdown files (auto-generated by GSD tools)

## Naming Conventions

**Files:**
- Page files: `page.tsx` (required by Next.js App Router)
- Component files: PascalCase (e.g., `Navigation.tsx`, `HeroSection.tsx`, `ProjectGrid.tsx`)
- Utility files: camelCase (e.g., `projects.ts`)
- CSS: `globals.css` (single global stylesheet)

**Directories:**
- Layout route groups: Lowercase folder names (e.g., `layout/`, `sections/`, `ui/`)
- Route segments: Lowercase folder names (e.g., `projects/`, `journal/`, `contact/`)
- Dynamic routes: Square brackets (e.g., `[slug]/`)

**Components:**
- Exported components: PascalCase function names matching file names
- Nested components (within a file): PascalCase (e.g., `ProjectCard` nested in `ProjectGrid.tsx`)

**Type Names:**
- Interfaces: PascalCase with suffix (e.g., `Project`, `ProjectCardProps`, `InsleyGridProps`)
- Enums/Unions: PascalCase (via type literals in TypeScript)

**CSS Variables:**
- Design tokens: Double-dash prefix, kebab-case (e.g., `--ink`, `--vellum`, `--stone`, `--font-cormorant`)
- Utility classes: Kebab-case (e.g., `.eyebrow`, `.section-label`, `.btn-primary`, `.grid-bg`)

## Where to Add New Code

**New Feature (New Project Type or Status):**
- Edit: `src/lib/projects.ts` — add type to union (`'NewType'`) and update `Project` interface if new fields needed
- No component changes required unless UI differs

**New Page/Route:**
- Create: `src/app/new-route/page.tsx` — use `'use client'` for client-side rendering
- Import: Navigation component from `src/components/layout/Navigation.tsx`
- Compose: Use existing section components from `src/components/sections/`
- Style: Use CSS variables and inline `style` props

**New Section Component:**
- Create: `src/components/sections/NewSection.tsx` — follow pattern from existing sections
- Include: `'use client'` directive, Framer Motion imports for animation
- Return: JSX with inline styles and CSS variables from `globals.css`
- Export: Default export matching file name (PascalCase)

**New UI Primitive:**
- Create: `src/components/ui/NewPrimitive.tsx`
- Pattern: Pure presentational component, likely with `'use client'` if using state/effects
- Use: CSS variables for colors, inline styles for layout

**New Section in Home/Detail:**
- Create: `src/components/sections/NewSection.tsx` (see above)
- Import: In target page (`src/app/page.tsx` or `src/app/projects/[slug]/page.tsx`)
- Integrate: Add to JSX between existing sections (order matters for visual flow)

**New Utility Function:**
- Create: `src/lib/new-utility.ts` or add to `src/lib/projects.ts`
- Export: Named or default export
- Use: Import where needed

**New Design Token:**
- Edit: `src/styles/globals.css` — add CSS variable in `:root` block
- Use: Reference via `var(--token-name)` in inline styles or class definitions
