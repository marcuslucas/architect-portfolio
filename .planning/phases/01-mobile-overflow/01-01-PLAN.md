- Inline `style` props dominate — fixes go into embedded `<style>` tags or globals.css media queries
- The `nav[style]` selector in Navigation's embedded style is unreliable: React renders inline styles as the `style` attribute but the attribute-contains selector `[style*="padding-left: 56px"]` will match raw attribute strings, which may differ from React's serialization. - `body { overflow-x: hidden }` already exists in globals.css but `html` is not constrained — add it there
- Do NOT touch any component's desktop layout, only add/fix mobile overrides

```css
}
```
Remove the old `nav[style]` line. Do NOT change any other styles or layout.
